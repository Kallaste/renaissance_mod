    • The ModLauncher

    The ModLauncher was developed to install a single modification (short: mod) in The Guild 2 - Renaissance. When doing so, the original files and saves are backed up in a special folder before the modified files are installed and played.
    If the original version of the game is to be restored or another mod installed, any saves are also backed up with the mod so that they aren’t lost.

    [ back ]

    How the ModLauncher works

    1) Installing a mod:
    To install a mod, unpack it in the following folder:

    TG2REN installation path/mods/

    Ensure that the downloaded mod is compatible with the ModLauncher (mod for Patch 4.2).
    The structure of the mod should be mod name/ folder structure TG2REN.

    2) Activate mod for playing the game:
    Once you have unpacked your mod, open the ModLauncher (after installation, the patch offers you the option of creating a desktop link to the ModLauncher). Click "Manage Modification", you will then see an overview box, in which all the available mods are listed (i.e. those you have extracted correctly).

    IMPORTANT: You can only ever activate one mod at a time in the game!

    Select the mod you wish to activate. Information about this mod will now appear in the ModLauncher. To activate the mod, click "Apply". The ModLauncher will now overwrite all the files for the selected mod in the game folder. Then click "Back" and then "Start the Game" in the main menu of the ModLauncher.
    Your Guild game will now start as usual and your mod is active.

    3) Deactivate mod:
    To deactivate a mod, open the ModLauncher, click "Manage Modification", select the active mod and then click "Reset to Original Version". The ModLauncher will now restore all original files.

    [ back ]

    Structuring modifications (for modders)

    Only a few steps are required to make your existing modifications compatible to ModLauncher. Please always ensure that you synchronise the file versions of your mods with that of Patch 4.2.
    To make it easier for you, we’ve included the "HotTeaMod" with the patch. Here you can look at the structures of the different files.

    IMPORTANT: Never use the following characters as they are XML control characters! < > & " ' =

    1) Structure of a mod:
    It is important to use the correct folder structure. The original "The Guild 2 Renaissance" folder structure is used for a mod.
    Example: Installation directory of TG2REN/mods/YOUR MOD NAME/Structure directory of TG2REN.
    Additionally, 3 new files are required: data.xml, description.html and text.xml, which must be located in the root directory of your mod.

    2) Structure of data.xml:
    As a template, it is recommended to edit the data.xml file from the example mod as follows.

    • Author: That’s you :)
    • Version: Enter the version number of your mod here.
    • Title: Enter the title of your mod here.
    • File: Files that are to be completely replaced, changed or added by your mod are placed in a file tag.
      • This step has to be completed for every file in your mod, otherwise the ModLauncher won’t know what to do.
    • Path: Here, enter the correct path to the file, which should be replaced or modified by the ModLauncher.
      • Example: .\mods\NAME OF YOUR MOD\CORRECT FOLDER STRUCTURE\YOUR MODIFIED FILE INCL. EXTENSION
      \mods\HotTeaMod\Textures\Hud\Buttons\btn_UseHotTea.tga
    • State: Below each file tag you will find a state tag. This tag tells the ModLauncher how to handle the specified file.
      • The following state tags can be used: "new" (if a completely new file should be added, i.e. one that currently doesn’t exist) and "changed" (if a file already exists but has been edited for your mod).
      NOTE: The original files, which are modified with the changed tag, are saved in the mods/Origin folder.
    3) Structure of description.html:
    As a template, it is recommended to edit the description.html file from the example mod.
    The description.html file is opened in the ModLauncher when a mod is selected in the list. This should contain all the information about the mod.
    The file is in html and can be very easily edited. As can be seen from the example file, the description.html file can also be set up in several different languages.

    4) Structure of text.xml:
    As a template, it is recommended to copy the text.xml file from the example mod and to adapt it to your mod.
    The text.xml can be set up in several languages. The ModLauncher recognises the chosen language version of The Guild 2 Renaissance (found in the config.ini file) and installs this directly from the text.xml file. If the selected language is not available in the text.xml file, English is automatically used. If English is also unavailable, the first language listed is used.
    The text.dbt file from The Guild 2 Renaissance does not need to be changed. All texts added or changed by the text.xml file will be adapted by the ModLauncher.

    • ID: The ID of the text, which is assigned by the ModLauncher in the text.dbt file (must be unique, except for the "changed" state, as the ID of the original entry is used)
      • IMPORTANT: Text IDs must be greater than 18000 so that your mod remains compatible for future patches :).
    • Label: The unique text label, which is used as an identifier for the text in the game, e.g.: _ITEM_HerbTea_NAME_+0.
    • Text language="LANGUAGE OF THE TEXT e.g. "english" for English": Enter your text here to be associated with the above-mentioned label.
      • NOTE: The Guild 2 Renaissance Patch 4.2 has been localised in the following languages: English, German, French, Spanish, Italian, Polish.
    • State:The state tag indicates whether existing text is to be edited, or if completely new text is to be added. The tags "new" (for new texts) and "changed" (for texts that are to be changed) can be used.

    • Der ModLauncher

    Der ModLauncher wurde entwickelt, um eine einzelne Modifikation (kurz: Mod) in Die Gilde 2 - Renaissance zu installieren. Dabei werden die originalen Dateien und die Spielstände in einem speziellen Ordner gesichert, bevor die modifizierten Dateien installiert bzw. eingespielt werden.
    Wenn die originale Version des Spiels wieder hergestellt bzw. wenn eine andere Mod installiert werden soll, werden auch etwaige Spielstände mit der Mod gesichert, so dass diese nicht verloren gehen.

    [ zurück ]

    Wie der ModLauncher funktioniert

    1) Mod installieren:
    Um eine Mod zu installieren, entpackt diese in folgenden Ordner:

    TG2REN Installationspfad/mods/

    Achtet darauf das die runtergeladene Mod mit dem ModLauncher kompatibel ist (Mod für Patch 4.2).
    Der Aufbau der Mod sollte Modname/ Ordnerstruktur TG2REN sein.

    2) Mod zum Spielen Spiel aktivieren:
    Habt Ihr eure Mod entpackt, öffnet ihr den ModLauncher (Der Patch bietet euch am Ende der Installation die Möglichkeit, eine Desktopverknüpfung vom ModLauncher anzulegen). Klickt auf "Manage Modification", anschließend gelang ihr in ein Übersichtsfeld, in welchem alle Mods aufgelistet werden, die euch zur Verfügung stehen (ihr also korrekt entpackt habt).

    WICHTIG: Ihr könnt immer nur einen Mod gleichzeitig im Spiel aktivieren!

    Wählt die Mod aus, welche ihr aktivieren wollt. Es erscheinen nun Infos zu diesem Mod im ModLauncher. Um die Mod zu aktivieren, klickt auf "Apply". Der ModLauncher überschreibt jetzt alle Dateien von der ausgewählten Mod in den Spielordner. Anschließend klickt ihr auf "Back" und im Hauptmenü des ModLaunchers auf "Start the Game".
    Euer Gilde-Spiel startet nun wie gewohnt und eure Mod ist aktiv.

    3) Mod deaktivieren:
    Um eine Mod zu deaktivieren, öffnet Ihr den ModLauncher, klickt auf "Manage Modification", wählt die aktive Mod aus und klickt anschließend auf "Reset to Original Version". Der ModLauncher stellt nun alle Orginaldateien wieder her.

    Aufbau von Modifikationen (für Modder)

    Um eure bestehenden Modifikationen "ModLauncher-kompatibel" zu machen, sind nur ein paar wenige Handgriffe nötig. Achtet bitte stets darauf, dass ihr die Dateiversionen eurer Mods mit denen von Patch 4.2 abgleicht.
    Um es euch einfacher zu machen, haben wir dem Patch den "HotTeaMod" beigepackt. Hier könnt ihr euch die Strukturen der verschiedenen Dateien ansehen.

    WICHTIG: Niemals die folgenden Zeichen verwenden, da dies Steuerzeichen von XML sind! < > & " ' =

    1) Aufbau eines Mods:
    Es ist wichtig, die korrekte Ordnerstruktur zu verwenden. Für eine Mod wird die originale "Die Gilde 2 Renaissance" Ordnerstruktur verwendet.
    Beispiel: Installationsverzeichnis von TG2REN/mods/EUER MODNAME/Strukturverzeichnis von TG2REN.
    Zusätzlich werden 3 neue Dateien benötigt: data.xml, description.html und text.xml, welche sich im Hauptverzeichnis eurer Mod befinden müssen.

    2) Aufbau der data.xml:
    Als Vorlage empfielt es sich, die aus der Beispiel-Mod verwendete data.xml wie folgt zu bearbeiten.

    • Author: Das bist du :)
    • Version: Trage hier die Versionsnummer deiner Mod ein.
    • Titel: Trage hier den Titel deiner Mod ein.
    • File: In einen File-Tag werden Dateien gesetzt, welche komplett ersetzt, verändert oder neu hinzugefügt werden durch eure Mod.
      • Dieser Schritt muss für jede Datei aus eurer Mod durchgeführt werden, da der ModLauncher anderenfalls nicht weiß, was er machen soll.
    • Path: Tragt hier den korrekten Path zu der Datei an, welche durch den ModLauncher ersetzt oder modifiziert werden soll.
      • Beispiel: .\mods\NAME DEINES MOD\KORREKTE ORDNERSTRUKTUR\DEINE VERÄNDERTE DATEI INKL. ENDUNG
      \mods\HotTeaMod\Textures\Hud\Buttons\btn_UseHotTea.tga
    • State: Unter jedem File-Tag findet ihr einen State-Tag. Dieser Tag sagt dem ModLauncher, wie er das darüber angegebene File behandeln soll.
      • Folgende State-Tag können verwendet werden: "new" (Wenn eine Datei komplett neu hinzugefügt wird, sie also bisher nicht existiert.) und "changed" (Wenn eine Datei bereits existiert, jedoch von dir für deine Mod geändert wurde.)
      HINWEIS: Die Orginaldateien, welche mit dem Tag Changed veränderten werden, werden im Ordner mods/Origin gespeichert.
    3) Aufbau der description.html:
    Als Vorlage empfiehlt es sich, die aus der Beispiel-Mod verwendete description.html zu bearbeiten.
    Die description.html wird im ModLauncher immer dann aufgerufen, wenn eine Mod in der Liste ausgewählt wird. Hier sollte man alle Informationen über die Mod erhalten.
    Die Datei ist in html aufgebaut und kann sehr leicht bearbeitet werden. Wie der Beispieldatei entnommen werden kann, kann die description.html auch mehrsprachig aufgebaut werden.

    4) Aufbau der text.xml:
    Als Vorlage empfiehlt es sich, die aus der Beispiel-Mod verwendete text.xml zu kopieren und an eure Mod anzupassen.
    Die text.xml kann mehrsprachig aufgebaut werden. Der ModLauncher erkennt die eingestellte Sprachversion von Die Gilde 2 Renaissance (zu finden in der config.ini) und installiert diese direkt aus der text.xml. Ist die eingestellte Sprache in der text.xml nicht vorhanden, wird automatisch Englisch verwendet. Ist Englisch ebenso nicht vorhanden, wird die zuerst aufgelistete Sprache verwendet.
    Die text.dbt aus Die Gilde 2 Renaissance muss nicht verändert werden. Alle durch die text.xml hinzugefügten oder veränderten Texte werden durch den ModLauncher angepasst.

    • ID: Die ID des Textes, welcher durch den ModLauncher in der Text.dbt vergeben wird (Muss einzigartig sein, außer beim State "changed", da muss die ID des originalen Eintrages verwendet werden)
      • WICHTIG: Text ID's müssen größer als 18000 sein, damit eure Mod auch für zukünftige Patches kompatibel bleibt :).
    • Label: Das eindeutige Textlabel, welches als Identifikator des Textes im Spiel verwendet wird, z.B.: _ITEM_HerbTea_NAME_+0.
    • Text language="SPRACHE DES TEXTES z.B. "english" für Englisch": Hier schreibt ihr euren Text rein, der mit dem o.g. Label assoziiert werden soll.
      • HINWEIS: Die Gilde 2 Renaissance Patch 4.2 wurde in den folgenden Sprachen lokalisiert: english, german, french, spanish, italian, polish.
    • State:Der State-Tag gibt an, ob ein bestehender Text verändert wird, oder ein völlig neuer Text hinzugekommen ist. Es können die Tags "new" (für neue Texte) oder "changed" (für Texte, die geändert werden sollen) verwendet werden.

    • El ModLauncher

    El ModLauncher ha sido desarrollado para instalar una modificación (o mod) en The Guild 2 - Renaissance. Antes de instalarla y jugar utilizando los archivos modificados, se creará una copia de seguridad de los archivos originales y de las partidas guardadas en una carpeta especial.

    Cómo funciona el ModLauncher

    1) Instalar una modificación:
    Para instalar una modificación, debes descomprimirla en la siguiente carpeta:

    TG2REN installation path/mods/

    Asegúrate de que la modificación que se ha descargado es compatible con el ModLauncher (modificación para el parche 4.2).
    La estructura de la modificación debe ser la siguiente: nombre de la modificación/estructura de la carpeta TG2REN.

    2) Activar una modificación para jugar:
    Una vez descomprimida la modificación, abre el ModLauncher (tras la instalación, el parche te ofrece la opción de crear un enlace del ModLauncher en el escritorio). Luego, haz clic en "Gestionar modificación" y aparecerá una caja con un resumen de todas las modificaciones disponibles (es decir, las que hayas extraído correctamente).

    IMPORTANTE: ¡El juego solo permite activar una modificación a la vez!

    Selecciona la modificación que quieras activar y aparecerá en el ModLauncher toda la información sobre la misma. Para activarla, haz clic en "Aplicar". A continuación, el ModLauncher sobrescribirá todos los archivos de la modificación seleccionada en la carpeta del juego. Haz clic en "Atrás" y luego en "Iniciar el juego" desde el menú principal del ModLauncher.
    De este modo, se iniciará el juego de la forma habitual, pero con la modificación activa.

    3) Desactivar una modificación:
    Para desactivar una modificación, abre el ModLauncher, haz clic en "Gestionar modificación" y selecciona la modificación activa. A continuación, haz clic en "Restaurar versión original" para que el ModLauncher restaure todos los archivos originales.

    Estructurar las modificaciones (para modders)

    Para hacer que tus modificaciones sean compatibles con el ModLauncher, basta con seguir los siguientes pasos. No olvides sincronizar las versiones de los archivos de tus modificaciones con las del parche 4.2.
    Para tu comodidad, hemos incluido en el parche la "HotTeaMod", donde puedes observar las estructuras de los distintos archivos.

    IMPORTANTE: ¡No uses nunca los siguientes caracteres, ya que son caracteres de control de XML! < > & " ' =

    1) Estructura una modificación:
    Es importante que estructures las carpetas de la forma correcta. Para las modificaciones se utiliza la estructura de carpetas original de "The Guild 2 - Renaissance".
    Ejemplo: Directorio de instalación de TG2REN/mods/NOMBRE DE TU MODIFICACIÓN/Directorio de la estructura de TG2REN.
    Además, se necesitan 3 archivos nuevos: data.xml, description.html y text.xml, que deben colocarse en el directorio raíz de tu modificación.

    2) Estructura de data.xml:
    Te recomendamos que utilices el archivo data.xml de la modificación de ejemplo para editarlo, a modo de plantilla, tal como se indica.

    • Autor: Es decir, tú :)
    • Versión: Introduce aquí el número de versión de tu modificación.
    • Título: Introduce aquí el título de tu modificación.
    • Archivo: La etiqueta de archivo se asigna a los archivos que serán completamente sustituidos, editados o añadidos por tu modificación.
      • Debes completar este paso con todos los archivos de tu modificación, de lo contrario el ModLauncher no sabrá qué hacer.
    • Itinerario: Aquí debes introducir la ruta correcta del archivo, que será sustituido o modificado por el ModLauncher.
      • Ejemplo: .\mods\NOMBRE DE TU MODIFICACIÓN\ESTRUCTURA DE CARPETAS CORRECTA\TU ARCHIVO MODIFICADO CON LA EXTENSIÓN INCLUIDA
      \mods\HotTeaMod\Textures\Hud\Buttons\btn_UseHotTea.tga
    • Estado: Debajo de cada etiqueta de archivo encontrarás una etiqueta de estado que le indica al ModLauncher cómo gestionar el archivo especificado.
      • Se pueden utilizar las siguientes etiquetas de estado: "nuevo" (para un archivo totalmente nuevo, es decir, que actualmente no existe) y "editado" (para un archivo existente pero que ha sido editado por tu modificación).
      NOTA: Los archivos originales modificados con la etiqueta "editado" se guardan en la carpeta mods/Origin.
    3) Estructura de description.html:
    Te recomendamos que utilices el archivo description.html de la modificación de ejemplo para editarlo, a modo de plantilla.
    El archivo description.html se abre en el ModLauncher cuando seleccionas una modificación de la lista. Este archivo debe contener toda la información sobre la modificación.
    El archivo está en formato html y se puede editar fácilmente. Tal como puede verse en el archivo de ejemplo, el archivo description.html también se puede configurar en distintos idiomas.

    4) Estructura de text.xml:
    Te recomendamos que copies el archivo text.xml de la modificación de ejemplo, a modo de plantilla, y que lo adaptes a la tuya.
    El archivo text.xml se puede configurar en distintos idiomas. El ModLauncher reconoce la versión elegida en The Guild 2 - Renaissance (que se encuentra en el archivo config.ini) y la instala directamente desde el archivo text.xml. Si el idioma seleccionado no está disponible en el archivo text.xml, se utilizará el inglés automáticamente. Si el inglés tampoco estuviera disponible, se utilizaría el primer idioma de la lista.
    No es necesario editar el archivo text.dbt de The Guild 2 - Renaissance. El ModLauncher adaptará todos los textos añadidos o editados por el archivo text.xml.

    • ID: Es la ID del texto, asignada por el ModLauncher en el archivo text.dbt. Esta ID debe ser única (excepto en el estado "editado", ya que se utiliza la ID de la entrada original).
      • IMPORTANTE: Las ID de texto deben consistir en un número mayor que 18 000 para que tu modificación sea compatible con futuros parches :).
    • Marca: Es una marca de texto única que se utiliza como identificador para el texto del juego. Por ejemplo: _ITEM_HerbTea_NAME_+0.
    • Idioma del texto="IDIOMA DEL TEXTO, por ejemplo, "english" para el inglés": Introduce el texto aquí para que se asocie con la marca que hemos mencionado más arriba.
      • NOTA: El parche 4.2 de The Guild 2 - Renaissance ha sido traducido a los idiomas siguientes: inglés, alemán, francés, español, italiano e polacco.

    • Il ModLauncher

    Il ModLauncher è stato sviluppato per consentire l'installazione di una singola modifica (o "mod") all'interno di The Guild 2 - Renaissance. Scegliendo di installare una mod, i file originali e i salvataggi verranno copiati in una cartella speciale prima di applicare i file modificati.
    Se è necessario ripristinare la versione originale del gioco, o nel caso in cui venisse installata un'altra mod, anche tutti i salvataggi verranno copiati insieme alla mod affinché non vadano persi.

    [ indietro ]

    Come funziona il ModLauncher

    1) Installare una mod:
    Per installare una mod, estraine il contenuto nella seguente cartella:

    percorso di installazione di TG2REN/mods/

    Assicurati che la mod scaricata sia compatibile con il ModLauncher (introdotto con la patch 4.2).
    La struttura della mod deve essere la seguente: nome della mod/struttura cartella TG2REN.

    2) Attivare la mod all'interno del gioco:
    Una volta estratta la mod, apri il ModLauncher (al termine dell'installazione, la patch ti permette di creare un collegamento al ModLauncher sul desktop). Clicca su "Gestisci modifiche" per visualizzare una finestra con l'elenco di tutte le mod disponibili (es: quelle estratte correttamente).

    IMPORTANTE: è possibile attivare una sola mod alla volta durante il gioco!

    Seleziona la mod che vuoi attivare. Le informazioni a riguardo verranno visualizzate nel ModLauncher. Per attivare la mod, clicca su "Applica". Il ModLauncher sovrascriverà tutti i file per la mod selezionata all'interno della cartella dove è installato il gioco. A seguire, clicca su "Indietro" e successivamente su "Avvia il gioco" nel menu principale del ModLauncher.
    The Guild verrà avviato normalmente con la mod installata attiva.

    3) Disattivare le mod:
    Per disattivare una mod, apri il ModLauncher, clicca su "Gestisci modifiche", seleziona la mod attiva e clicca su "Ripristina versione originale". Il ModLauncher provvederà dunque a ripristinare tutti i file originali.

    Modifiche alla struttura (per i modder)

    Bastano pochi passaggi per rendere le tue modifiche esistenti compatibili con il ModLauncher. Ricordati sempre di sincronizzare le versioni dei file delle tue mod con quelle della patch 4.2.
    Per facilitarti il compito, abbiamo incluso la "HotTeaMod" all'interno della patch. Essa permette di dare un'occhiata alla struttura dei diversi file.

    IMPORTANTE: non utilizzare mai i seguenti caratteri, in quanto si tratta di caratteri di controllo XML! < > & " ' =

    1) Struttura di una mod:
    È importante usare la struttura corretta per la cartella. Le mod utilizzano la struttura originale della cartella di "The Guild 2 Renaissance".
    Esempio: la cartella di installazione di TG2REN/mods/NOME DELLA TUA MOD/struttura cartella di TG2REN.
    È necessario inoltre aggiungere 3 nuovi file, data.xml, description.html e text.xml, nel percorso principale della mod.

    2) Struttura del file data.xml:
    Come riferimento, si consiglia di modificare il file data.xml della mod di esempio nella seguente maniera.

    • Autore: tu :)
    • Versione: qui inserisci la versione della tua mod.
    • Titolo: qui inserisci il titolo della tua mod.
    • File: i file che vanno completamente sostituiti, modificati o aggiunti dalla tua mod vengono messi in un file tag.
      • Questo passaggio va eseguito per ogni file della mod, altrimenti il ModLauncher non funzionerà.
    • Percorso: qui inserisci il percorso corretto del file che deve essere sostituito o modificato dal ModLauncher.
      • Esempio: .\mods\NOME DELLA TUA MOD\STRUTTURA CORRETTA DELLA CARTELLA\IL TUO FILE MODIFICATO COMPRESA ESTENSIONE
      \mods\HotTeaMod\Textures\Hud\Buttons\btn_UseHotTea.tga
    • Stato: sotto ogni file tag è presente un tag di stato che indica al ModLauncher come gestire il file in questione.
      • È possibile usare i seguenti tag di stato: "nuovo" (nel caso in cui sia necessario aggiungere un file completamente nuovo, ad esempio uno che non esiste) e "modificato" (se il file esiste ma è stato modificato dalla tua mod).
      NOTA: i file originali che sono stati modificati e a cui è stato assegnato il tag modificato vengono salvati nella cartella mods/Origin.
    3) Struttura del file description.html:
    Come riferimento, si consiglia di modificare il file description.html della mod di esempio.
    Il file description.html viene aperto nel ModLauncher dopo aver selezionato una mod tra quelle presenti nell'elenco. Esso contiene tutte le informazioni sulla mod.
    Il file è in formato html ed è facilmente modificabile. Come si può vedere nel file di esempio, il file description.html può essere impostato anche in diverse lingue.

    4) Struttura del file text.xml:
    Come riferimento, si consiglia di copiare il file text.xml dalla mod di esempio e adattarlo alla tua mod.
    Il file text.xml può essere impostato in diverse lingue. Il ModLauncher riconosce la versione della lingua scelta di The Guild 2 Renaissance (situata all'interno del file config.ini) e provvede a installarla direttamente dal file text.xml. Se la lingua selezionata non è disponibile nel file text.xml, verrà usato automaticamente l'inglese. Se anche la lingua inglese non dovesse essere disponibile, verrà utilizzata la prima tra quelle presenti nell'elenco.
    Il file text.dbt di The Guild 2 Renaissance non necessita di modifiche. Tutti i testi aggiunti o modificati dal file text.xml verranno adattati automaticamente dal ModLauncher.

    • ID: l'ID del testo, assegnato dal ModLauncher nel file text.dbt (deve essere unico, tranne per lo stato "modificato", in quanto viene usato l'ID della versione originale)
      • IMPORTANTE: gli ID del testo devono essere maggiori di 18000, in modo da garantire la compatibilità della tua mod con le patch future:).
    • Etichetta: l'etichetta unica del testo, utilizzata come identificatore per il testo all'interno del gioco. Es.: _ITEM_HerbTea_NAME_+0.
    • Lingua testo="LA LINGUA DEL TESTO es. "english" per l'inglese": inserisci qui il tuo testo per associarlo all'etichetta di cui sopra.
      • NOTA: la patch 4.2 di The Guild 2 Renaissance è stata localizzata nelle seguenti lingue: inglese, tedesco, francese, spagnolo, italiano, polaco.
    • Stato:il tag stato indica se il testo esistente va modificato o se è necessario aggiungere del testo completamente nuovo. Si possono usare i tag "nuovo" (per i testi nuovi) e "modificato" (per i testi da modificare).

    Polski - Instrukcja

     Witaj w aplikacji The Guild 2 - Renesans ModLauncher

    Spis treści

  • Aplikacja ModLauncher
  • Jak działa ModLauncher
  • Przygotowywanie struktury modyfikacji (dla modderów)

    • Aplikacja ModLauncher

    ModLauncher został opracowany w celu ułatwienia instalacji pojedynczych modyfikacji dla gry The Guild 2 - Renesans. Podczas procesu instalacji tworzone są kopie zapasowe oryginalnych plików oraz zapisów gier, a dopiero później instalowane są nowe i/lub zmienione pliki.
    W przypadku przywracania oryginalnej wersji gry lub instalacji innej modyfikacji, wszelkie zapisy również są archiwizowane wraz z daną modyfikację, zatem nic nie powinno zginąć.

    [ wróć ]

    Jak działa ModLauncher

    1) Instalacja modyfikacji:
    W celu instalacji modyfikacji wypakuj ją do następującego folderu:

    Folder instalacji The Guild 2 - Renesans/mods/

    Upewnij się, iż pobrana modyfikacja jest zgodna z aplikacją ModLauncher (modyfikacja dla gry w wersji 4.2).
    Struktura modyfikacji powinna wyglądać następująco: Nazwa modyfikacji/Struktura folderów gry The Guild 2 - Renesans.

    2) Aktywacja modyfikacji:
    Kiedy już wypakujesz modyfikację zgodnie z pkt. 1, uruchom ModLauncher (podczas instalacji aktualizacji możesz zgodzić się na utworzenie skrótu do aplikacji ModLauncher na swoim pulpicie). Kliknij "Manage Modification" (zarządzaj modyfikacjami), a zobaczysz okno z listą wszystkich dostępnych modyfikacji (np. tych, które prawidłowo wypakowałeś).

    WAŻNA INFORMACJA: Można aktywować tylko jedną modyfikację w danym czasie!

    Wybierz modyfikację, którą chcesz aktywować. Informacja o niej pojawi się w oknie aplikacji ModLauncher. W celu aktywacji modyfikacji kliknij "Apply" (zastosuj). ModLauncher nadpisze wówczas wszystkie niezbędne pliki w katalogu gry. Następnie kliknij "Back" (wróć) i w końcu "Start the Game" (uruchom grę) z poziomu głównego menu aplikacji ModLauncher.
    Gra uruchomi się, a wybrana modyfikacja będzie już aktywna.

    3) Dezaktywacja modyfikacji:
    W celu dezaktywacji modyfikacji uruchom ModLauncher, kliknij "Manage Modification" (zarządzaj modyfikacjami), wybierz aktywną modyfikację i kliknij "Reset to Original Version" (przywróć wersję oryginalną). ModLauncher przywróci oryginalne pliki gry i usunie pliki dodane przez modyfikację.

    [ wróć ]

    Przygotowywanie struktury modyfikacji (dla modderów)

    Wystarczy kilka kroków, aby przystosować Twoją modyfikację do obsługi przez aplikację ModLauncher. Ważne, by pliki modyfikacji były zgodne z plikami gry z wersji 4.2.
    W aktualizacji 4.2 zawarliśmy przykładową modyfikację "HotTeaMod", która posłuży nam za przykład. Możesz zapoznać się ze strukturą poszczególnych plików ten modyfikacji, co ułatwi Ci pracę nad własną modyfikacją.

    WAŻNA INFORMACJA: Nigdy nie stosuj w treści znaków zarezerwowanych dla struktury XML! Tymi znakami są: < > & " ' =

    1) Struktura modyfikacji:
    Bardzo istotną kwestią podczas przygotowywania modyfikacji jest zachowanie prawidłowej struktury folderów, zgodnej ze strukturą katalogów gry "The Guild 2 - Renesans".
    Przykład: Katalog instalacji The Guild 2 - Renesans/mods/NAZWA TWOJEJ MODYFIKACJI/Struktura zgodna ze strukturą The Guild 2 - Renesans.
    Dodatkowo modyfikacja wymaga 3 nowych plików. Są to: data.xml, description.html oraz text.xml, które muszą znajdować się w głównym katalogu Twojej modyfikacji.

    2) Struktura pliku data.xml:
    Jako postawę zaleca się wykorzystanie pliku, który znajduje się w modyfikacji przykładowej.

    • Author: To ty, autor :)
    • Version: W to pole wprowadź numer wersji swojej modyfikacji.
    • Title: W to pole wprowadź tytuł swojej modyfikacji.
    • File: W tym miejscu wprowadź listę plików, które mają zostać zastąpione, zmodyfikowane bądź dodane przez Twoją modyfikację wraz z określeniem ich stanu - opisane poniżej.
      • Ten krok musi zostać powtórzony dla każdego pliku Twojej modyfikacji, w innym wypadku ModLauncher nie będzie wiedział jak postąpić z plikami.
    • Path: W tym miejscu wprowadź ścieżkę dla danego pliku, który powinien zostać zastąpiony bądź zmodyfikowany przez aplikację ModLauncher.
      • Przykład:
      .\mods\NAZWA MODYFIKACJI\PRAWIDŁOWA STRUKTURA KATALOGÓW\TWOJA WERSJA PLIKU WRAZ Z ROZSZERZENIEM
      .\mods\HotTeaMod\Textures\Hud\Buttons\btn_UseHotTea.tga
    • State: Poniżej każdego pliku znajdziesz oznaczenie jego stanu. Ten parametr przekazuje aplikacji ModLauncher jak postąpić z danym plikiem.
      • Dozwolone są następujące parametry stanu: "new" (dla plików całkowicie nowych, np. dla tych, które nie istnieją w grze) oraz "changed" (dla plików, które istnieją w grze, ale mają zostać zmienione przez Twoją modyfikację).
      UWAGA: Oryginalne pliki, które są zmieniane przez modyfikację (oznaczone odpowiedni parametrem stanu), są zapisywane w folderze "mods/Origin".
    3) Struktura pliku description.html:
    Jako podstawę zaleca się wykorzystanie pliku description.html, który znajduje się w modyfikacji przykładowej.
    Plik description.html jest otwierany przez ModLauncher, gdy dana modyfikacja jest wybrana z listy. Powinien zawierać wszystkie informacje związane z modyfikacją.
    Plik zapisany jest w formacie html i może być edytowany bez najmniejszych problemów np. za pomocą edytora tekstu (Notatnik czy Notepad++). Jak widać w przykładowym pliku, description.html może zawierać opisy w kilku językach.

    4) Struktura pliku text.xml:
    Jak podstawę zaleca się wykorzystanie kopii pliku text.xml z modyfikacji przykładowej i dostosowanie go do swojego projektu.
    Plik text.xml może zawierać kilka języków. ModLauncher rozpoznaje wybrany język gry The Guild 2 - Renesans (określany na podstawie pliku config.ini), a następnie podczas instalacji wprowadza zmiany dla danego języka, które znajdzie w pliku text.xml. Jeśli wybrany język nie jest dostępny w pliku text.xml, ModLauncher automatycznie wybierze język angielski. W przypadku braku języka angielskiego, wykorzystany zostanie pierwszy dostępny język z pliku.
    Plik text.dbt z gry The Guild 2 - Renesans, czyli główny plik z tekstami, nie musi być modyfikowany. Wszystkie teksty dodane lub zmienione, które są zawarte w pliku text.xml, będą odpowiednio przeniesione do głównego pliku tekstowego przez aplikację ModLauncher.

    • ID: Numer ID tekstu, który ma zostać przypisany przez ModLauncher w pliku text.dbt (dla nowych tekstów numer ID musi być niepowtarzalny, dla tekstów oznaczonych parametrem stanu "changed", numer ID musi być zgodny z oryginalnym wpisem)
      • WAŻNA INFORMACJA: ID tekstów nowych muszą być wyższe nic 18000, gdyż niższe wartości zarezerwowane są dla gry i przyszłych aktualizacji :).
    • Label: Unikatowa etykieta tekstu, która jest przyporządkowana treści wyświetlanej w grze, np.: _ITEM_HerbTea_NAME_+0.
    • Text language=Język tekstu np. "polish" dla języka polskiego: Tu wpisz swój tekst, który będzie powiązany z etykietą opisaną powyżej.
      • UWAGA: Aktualizacja 4.2 dla gry The Guild 2 - Renesans została przygotowana dla następujących języków: angielski, niemiecki, francuski, hiszpański, włoski oraz polski.
    • State: Parametr stanu określa czy dany tekst jest nowy, czy jest tekstem zmodyfikowanym występującym już w grze. Wartość "new" oznacza nowy teksty, a wartość "changed" oznacza teksty, który zmienia już istniejący w grze.
